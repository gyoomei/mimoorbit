Hermes Agent

Linux / macOS / WSL2 (Windows). Universal AI agent CLI with skills, memory, and multi-platform delivery.

# 1. Install
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
source ~/.bashrc
hermes --version

# 2. Setup with Xiaomi MiMo (predefined provider)
hermes setup
#   → Quick Setup → Provider: Xiaomi MiMo
#   → API Key: sk-xxxxx (or tp-xxxxx for Token Plan)
#   → Base URL: https://api.xiaomimimo.com/v1
#   → Default model: mimo-v2.5-pro

# 3. Or use custom provider (Token Plan)
hermes config set model.provider custom
hermes config set model.base_url https://token-plan-cn.xiaomimimo.com/v1
hermes config set model.api_key tp-xxxxx
hermes config set model.default mimo-v2.5-pro

# 4. Verify and launch
hermes doctor
hermes            # CLI mode
hermes --tui      # TUI mode

OpenClaw

Multi-channel AI bot framework. Requires Node.js 22+. Token Plan needs manual config (Method 2 below).

# 1. Install (macOS / Linux)
curl -fsSL https://openclaw.ai/install.sh | bash
# Windows PowerShell:
# iwr -useb https://openclaw.ai/install.ps1 | iex

# 2. Method 1 — Interactive wizard (Pay-as-you-go only)
openclaw onboard --install-daemon
#   → QuickStart → Provider: Xiaomi → API Key → channels → skills → done
#   → Test: openclaw tui

# 3. Method 2 — Edit ~/.openclaw/openclaw.json (Token Plan supported)
{
  "models": {
    "mode": "merge",
    "providers": {
      "xiaomi-coding": {
        "baseUrl": "https://token-plan-cn.xiaomimimo.com/v1",
        "apiKey": "tp-xxxxx",
        "api": "openai-completions",
        "models": [
          { "id": "mimo-v2.5-pro", "name": "mimo-v2.5-pro", "reasoning": true,
            "input": ["text"], "contextWindow": 1048576, "maxTokens": 32000 },
          { "id": "mimo-v2.5", "name": "mimo-v2.5", "reasoning": true,
            "input": ["text", "image"], "contextWindow": 262144, "maxTokens": 32000 }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": { "primary": "xiaomi-coding/mimo-v2.5-pro" },
      "models": { "xiaomi-coding/mimo-v2.5": {}, "xiaomi-coding/mimo-v2.5-pro": {} }
    }
  }
}
Note: For Token Plan, delete the "auth" field and rename provider to anything other than xiaomi (e.g. xiaomi-coding) to avoid collision with the preset gateway.

Python SDK · OpenAI-compatible

Drop-in OpenAI replacement. Just swap base_url and api_key.

pip install -U openai

# call.py
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["MIMO_API_KEY"],          # sk-xxxxx or tp-xxxxx
    base_url="https://api.xiaomimimo.com/v1"     # or token-plan-cn URL
)

resp = client.chat.completions.create(
    model="mimo-v2.5-pro",
    messages=[
        {"role": "system",
         "content": "You are MiMo, an AI assistant developed by Xiaomi."},
        {"role": "user", "content": "Explain what makes MiMo V2.5 unique."}
    ],
    max_completion_tokens=1024,
    temperature=1.0,
    top_p=0.95
)
print(resp.choices[0].message.content)

FAQ

The questions builders actually ask. Distilled from the official 100t.xiaomimimo.com docs.

When does the grant period end?▼
May 28, 2026 00:00 Beijing Time (UTC May 27, 16:00). 30-day window. Ends early if all 100T tokens are claimed.
What do I get if approved?▼
Either a Token Plan subscription (plug into Claude Code, Cursor, Hermes, OpenClaw, etc.) or pay-as-you-go bonus credits, depending on your application strength. Tier scales with how specific and detailed your project is.
Who can apply?▼
Open globally. Individual developers, teams, and enterprises. The bar is "real builders using AI" — quality of submission matters more than scale.
What makes a strong application?▼
List the actual AI tools you use (e.g. Hermes Agent, Claude Code, Cursor), name the underlying models, describe the project with concrete numbers (LOC, transactions, users), and link a live demo or repo. Specificity > vagueness.
Why does my chatbox here use Pollinations, not the real MiMo?▼
Pollinations.ai is a free, no-key OpenAI-compatible gateway used here so anyone can try it without signup. Once you receive your grant, swap the endpoint to https://api.xiaomimimo.com/v1 with your sk-xxxxx key for the actual MiMo V2.5 reasoning model with proper rate limits.
Email I used differs from my Xiaomi ID — what now?▼
Bind the application email to your Xiaomi ID at id.mi.com BEFORE you submit, OR register a new platform account with the email after approval. Mismatch is the #1 cause of grants failing to deliver.
